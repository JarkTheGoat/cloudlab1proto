LabVIEW

Front panel and block diagram

Question: Where does a LabVIEW program’s interface end and its logic begin?

The front panel is the user-facing interface containing controls and indicators. The block diagram contains terminals, wires, functions, structures, and dataflow. A useful diagnostic question is: “Is this element something the user manipulates or something the program executes?”

LabVIEW

Controls and indicators

A control is an input to the VI. An indicator displays an output. For the voltage-divider VI, Vin, R1, and R2 are controls; Vout is an indicator.

Check: Is a Stop button a control or indicator? It is a Boolean control because it supplies a value to the loop condition.
LabVIEW

VI and SubVI

A VI is a LabVIEW program. When a VI has a connector pane and is placed inside another block diagram, it functions as a SubVI. A future version of the lab could package the voltage-divider calculation as a reusable SubVI.

LabVIEW

Data types, arrays, and clusters

Numeric wires carry numbers, Boolean wires carry true/false values, and strings carry text. An array stores multiple values of the same type. A cluster groups related values and may combine different types.

Example: five measured voltages naturally form an array; a cluster could combine a student ID, the five voltages, and a completion Boolean.

LabVIEW

While loops, for loops, and shift registers

  • While loop: repeat until a stop condition is met.
  • For loop: repeat a known number of times.
  • Shift register: carry a value from one iteration to the next.

The required VI uses a while loop because it should continue recalculating until the user presses Stop.

Electronics

Voltage divider

Vout = Vin × R2 ÷ (R1 + R2)

R1 is between Vin and the output node. R2 is between the output node and ground. The formula is only meaningful when the circuit orientation and the definition of Vout are clear.

Common error: swapping R1 and R2 reverses the predicted trend.
Sensor

Photoresistor

A photoresistor, or light-dependent resistor, changes resistance with illumination. In the device used in this lab, resistance is expected to be higher in darkness and lower in brighter light. It is a passive resistive sensor, so a voltage divider is used to turn its change into a measurable voltage.

Measurement

Using a multimeter

Resistance mode

Power the circuit off and isolate the component or network when practical. Place the meter across the resistance being measured.

Voltage mode

Power the circuit on. Place the meter in parallel between the measurement node and reference ground.

Confirm lead sockets and mode before touching the circuit. Current mode is not used in this lab.

Hardware

Breadboard and trainer board

Rows of five holes are usually internally connected. The centre gap separates the two halves. Power rails run along the edges but may be split on some boards. A trainer board adds a controlled power supply and other built-in resources.

Always verify continuity and rail segmentation rather than assuming the board matches a picture.

Hardware

Resistor values, LED polarity, and current limiting

The divider uses 4.7 kΩ. The LED path uses 220 Ω to limit current. An LED is polarized: the anode and cathode must be oriented correctly. A reversed LED usually remains off; an LED without suitable current limiting may be damaged.

Arduino

Analog input versus digital output

An analog input converts a voltage range into a numerical reading. On the Arduino Uno used in this activity, analogRead() typically returns an integer from 0 to 1023. A digital output is commanded as LOW or HIGH.

A0 therefore receives the sensor’s varying voltage, while D7 commands the LED’s two-state action.

Arduino

Arduino C++ basics for this lab

  • setup() runs once after startup or reset.
  • loop() repeats continuously.
  • pinMode(pin, OUTPUT) configures a digital output.
  • analogRead(A0) reads the sensor voltage as a number.
  • digitalWrite(pin, HIGH/LOW) commands the output.
  • if/else selects one action based on a Boolean condition.
Simulation

Tinkercad Circuits

Tinkercad allows a circuit to be assembled, simulated, measured, and programmed before using physical hardware. Simulation is a model, not proof that a real circuit will work. Use it to test reasoning and discover wiring errors, then verify on the bench.

Engineering practice

Calculated versus measured values

A theoretical value assumes ideal components and controlled conditions. A measured value includes resistor tolerance, sensor variability, supply variation, meter accuracy, contact resistance, and changing illumination. Difference is evidence to interpret, not automatically a mistake.

percent difference = |measured − calculated| ÷ |calculated| × 100%
Engineering practice

A repeatable troubleshooting sequence

  1. State the expected behaviour and the observed behaviour.
  2. Remove power before changing wiring.
  3. Inspect supply voltage and common ground.
  4. Trace one signal path from source to output.
  5. Measure at intermediate nodes.
  6. Change one thing at a time.
  7. Retest and record what changed.
