Laboratory 1 · Foundations

LabVIEW, Tinkercad and Arduino Sensor Interfaces

Build the same engineering idea three ways: first as a LabVIEW calculation, then as a simulated sensor circuit, and finally as a real photoresistor system that turns an LED on in low light.

Estimated time: 3–4 hours Software: LabVIEW + Tinkercad Hardware: trainer board + multimeter Evidence: VI, quiz, tables, demonstrations

The guiding question

How can a physical change in light become a number, a decision, and an action?

  • Light changes resistance.
  • A voltage divider changes resistance into voltage.
  • An Arduino reads voltage as a number.
  • Code turns that number into an LED action.

Data types, arrays, and clusters

Data type determines what a wire carries. Arrays hold repeated values of one type; clusters group related values, possibly of different types.

While loops, for loops, and shift registers

A while loop repeats until a condition is met. A for loop repeats a known number of times. A shift register carries a value from one loop iteration to the next.

Stage 6 of 7 · ILO 6–7

Turn a sensor reading into an LED decision

Tinkercad Exercises 3–4

Connect the voltage-divider output to A0, add an LED to digital pin 7, and use a threshold to decide whether the LED should be HIGH or LOW.

Text-only wiring map

  1. Delete the multimeter.
  2. Connect J7 to Arduino A0.
  3. Place the LED at G19 and G20.
  4. Place a 220 Ω resistor from G14 to G18.
  5. Connect F18 to F19.
  6. Connect J14 to GND.
  7. Connect J20 to digital pin 7.
Check LED polarity. The current-limiting resistor protects the LED; reversing the LED may prevent it from lighting.
Arduino C++ sketch
int PRPin = 0;
int PR = 0;
int LEDPin = 7;
int lowLightLim = 400;

void setup()
{
  pinMode(LEDPin, OUTPUT);
}

void loop()
{
  PR = analogRead(PRPin);
  if (PR < lowLightLim) {
    digitalWrite(LEDPin, HIGH);
  }
  else {
    digitalWrite(LEDPin, LOW);
  }
}

Read the code as an argument

  1. Claim: pin 7 will control an output.
  2. Evidence: A0 is sampled with analogRead().
  3. Rule: if the value is below 400, write HIGH; otherwise write LOW.
  4. Repetition: loop() repeats the sensing and decision continuously.

Verify and troubleshoot

Run the simulation. Change the light level around the switching point. If the LED does not respond, inspect in this order: sensor value in the code, A0 connection, common ground, LED polarity, 220 Ω resistor path, pin 7 connection, and the comparison operator.
